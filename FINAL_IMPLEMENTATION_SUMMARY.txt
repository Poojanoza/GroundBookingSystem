Image Upload Flow:
  1. User selects image â†’ File path obtained from MediaStore
  2. File bytes read from device storage
  3. RequestBody created from bytes
  4. MultipartBody.Part created for form data
  5. Retrofit call: POST /api/grounds/upload-image
  6. Backend receives multipart file
  7. Backend stores in Supabase Storage (bucket: ground-images)
  8. Backend returns image URL
  9. App stores URL in memory
  10. Retrofit call: POST /api/grounds with image_url
  11. Backend creates ground record in database
  12. Ground available for users to book

Authentication:
  - Email/Password login
  - JWT token generated (7-day expiry)
  - Token stored in SharedPreferences
  - Bearer token in Authorization header for all API calls
  - Backend validates token on every request

Permissions:
  - READ_EXTERNAL_STORAGE (for image selection)
  - WRITE_EXTERNAL_STORAGE (for image cache)
  - INTERNET (for API calls)
  - ACCESS_NETWORK_STATE (for connectivity check)
